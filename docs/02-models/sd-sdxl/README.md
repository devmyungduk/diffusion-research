## SD 1.5 vs SDXL 비교

### 📊 주요 차이점

| 항목 | SD 1.5 | SDXL |
|------|--------|------|
| **해상도** | 512×512 기본 | 1024×1024 기본 |
| **품질** | 기본 | 향상됨 ⭐ |
| **VRAM** | 4GB+ | 8GB+ |
| **속도** | 빠름 ⚡ | 느림 |
| **파일 크기** | ~2GB | ~6GB |
| **프롬프트 이해** | 보통 | 더 좋음 |

### 💡 어떤 것을 선택할까?

**SD 1.5를 선택하세요:**
- VRAM이 4-6GB
- 빠른 생성 필요
- 512×512 크기로 충분

**SDXL을 선택하세요:**
- VRAM이 8GB+
- 고해상도 필요
- 더 나은 품질 원함

## 기본 사용법

### 🏗️ 워크플로우 구조

```
[Load Checkpoint]
    ├─→ MODEL → [KSampler]
    ├─→ CLIP → [CLIP Text Encode]
    └─→ VAE → [VAE Decode]

[Empty Latent Image] → [KSampler]
[KSampler] → [VAE Decode]
[VAE Decode] → [Save Image]
```

### 📝 단계별 가이드

**1. 모델 로드**
```
노드: Load Checkpoint
파일: realisticVision_v60.safetensors (예시)
```

**2. 프롬프트 작성**
```
Positive: a beautiful landscape, high quality
Negative: blurry, low quality, distorted
```

**3. 이미지 크기 설정**
- SD 1.5: 512×512
- SDXL: 1024×1024

**4. 생성!**
```
Queue Prompt 클릭
```

## 권장 설정

### 📊 SD 1.5 권장 설정

| 항목 | 값 | 이유 |
|------|-----|------|
| width × height | 512×512 | 최적 해상도 |
| steps | 25-30 | 품질/속도 균형 |
| cfg | 7-8 | 표준 |
| sampler | euler, dpmpp_2m | 안정적 |

### 📊 SDXL 권장 설정

| 항목 | 값 | 이유 |
|------|-----|------|
| width × height | 1024×1024 | 최적 해상도 |
| steps | 30-40 | 고품질 |
| cfg | 7-9 | 표준 |
| sampler | dpmpp_2m_karras | SDXL 최적화 |

### 🎨 해상도 가이드

**SD 1.5 권장 해상도:**
```
512×512   ✅ 최적
768×512   ✅ 와이드
512×768   ✅ 세로
1024×1024 ❌ 너무 큼 (품질 저하)
```

**SDXL 권장 해상도:**
```
1024×1024 ✅ 최적
1024×768  ✅ 가로
768×1024  ✅ 세로
512×512   ⚠️ 너무 작음
```

## 문제 해결

### 이미지가 이상해요

#### SD 1.5에서 1024×1024 생성 시
**문제:** 반복 패턴, 품질 저하  
**해결:** 512×512로 생성 후 Upscale

#### SDXL에서 512×512 생성 시
**문제:** 디테일 부족, 흐릿함  
**해결:** 1024×1024로 크기 변경

### 메모리 부족

**SD 1.5 (4GB VRAM):**
```
- 이미지 크기: 512×512로 제한
- Batch size: 1
- VAE: tiled VAE 사용
```

**SDXL (8GB VRAM):**
```
- 이미지 크기: 1024×1024로 제한
- Batch size: 1
- --lowvram 옵션
```

**장점:**
- 빠른 반복 테스트

**장점:**
- 고품질 결과
- 복잡한 프롬프트 이해
- 사실적 이미지

**추천 용도:**
- 최종 결과물
- 사실적 이미지
- 고해상도 필요 시
