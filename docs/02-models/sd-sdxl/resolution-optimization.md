## 🎯 가이드 개요

**대상 독자:** SDXL 모델로 고품질 이미지를 생성하고 싶은 사용자
**사전 이해:** 해상도(Width/Height) 설정법
**예상 소요 시간:** 10분

**핵심 내용:**
- SDXL이 학습된 공식 해상도 목록 (Buckets)
- 왜 1024x1024 외에 다른 비율을 쓰면 이미지가 깨지는지 원리
- 16:9, 21:9 등 원하는 비율을 안전하게 뽑는 방법

## 1. SDXL 해상도 개념

### 🎯 핵심 이해

**SDXL = 1024×1024 해상도에 최적화된 모델**

특정 해상도에서 최고 성능을 발휘하도록 학습되었습니다.

### 📊 SD 1.5 vs SDXL 비교

| 모델 | Native 해상도 | 학습 데이터 | 권장 범위 |
|------|--------------|------------|-----------|
| **SD 1.5** | 512×512 | 512px 중심 | 448~768px |
| **SDXL** | 1024×1024 | 1024px 중심 | 832~1216px |

### 💡 왜 특정 해상도인가?

```
학습 시 사용한 해상도 = 최적 성능 해상도

다른 해상도 사용 시:
  ✅ 작동은 함
  ⚠️ 품질 저하 가능
  ⚠️ 구도 왜곡 가능
  ⚠️ 아티팩트 발생 가능
```

## 2. Nearest SDXL Resolution

### 🎯 기능 개요

**Nearest SDXL Resolution = 가장 가까운 권장 해상도로 자동 조정**

사용자가 지정한 해상도를 SDXL의 권장 해상도 중 가장 가까운 값으로 변환합니다.

### 📊 작동 방식

```
사용자 입력: 800×1200
    ↓
시스템 계산: 가장 가까운 권장 해상도는?
    ↓
자동 조정: 832×1216
    ↓
모델 실행: 최적 품질 보장
```

### 💡 목적

| 목적 | 설명 |
|------|------|
| **품질 보장** | 모델의 학습 해상도와 일치 |
| **아티팩트 방지** | 비표준 해상도의 왜곡 최소화 |
| **계산 효율** | 최적화된 해상도로 빠른 처리 |

### ⚙️ 사용 여부

**활성화 (기본값):**
```
✅ 자동으로 최적 해상도 조정
✅ 품질 안정성 보장
✅ 초보자 추천
```

**비활성화:**
```
⚠️ 정확한 해상도 필요시
⚠️ 특수한 작업
⚠️ 고급 사용자
```

## 3. 권장 해상도 목록

### 📊 SDXL 권장 해상도

#### 정사각형

| 해상도 | 비율 | 용도 |
|--------|------|------|
| **1024×1024** | 1:1 | 프로필, SNS, 표준 |

#### 가로 (Landscape)

| 해상도 | 비율 | 용도 |
|--------|------|------|
| **1152×896** | 1.29:1 | 풍경, 와이드 |
| **1216×832** | 1.46:1 | 영화, 파노라마 |
| **1344×768** | 1.75:1 | 초와이드 |
| **1536×640** | 2.4:1 | 배너 |

#### 세로 (Portrait)

| 해상도 | 비율 | 용도 |
|--------|------|------|
| **896×1152** | 0.78:1 | 인물, 세로 |
| **832×1216** | 0.68:1 | 전신, 타이트 |
| **768×1344** | 0.57:1 | 모바일 세로 |
| **640×1536** | 0.42:1 | 스토리 형식 |

### 📏 해상도 자동 조정 예시

| 입력 해상도 | → | 조정된 해상도 | 차이 |
|-------------|---|--------------|------|
| 800×1200 | → | 832×1216 | 소폭 조정 |
| 1000×1000 | → | 1024×1024 | 미세 조정 |
| 1280×720 | → | 1344×768 | 가까운 비율 |
| 512×512 | → | 1024×1024 | ⚠️ 2배 확대 |

## 4. 실전 활용 가이드

### 🎯 시나리오별 권장 설정

#### Scenario 1: SNS 포스트

**목표:** 정사각형 썸네일  
**권장 해상도:** 1024×1024

```
이유:
  ✅ Instagram/Twitter 최적
  ✅ 빠른 생성
  ✅ 표준 품질
```

---

#### Scenario 2: 배경 화면

**목표:** 와이드 풍경  
**권장 해상도:** 1344×768

```
이유:
  ✅ 16:9 비율과 유사
  ✅ 데스크톱 배경 적합
  ✅ 풍경 구도 최적
```

---

#### Scenario 3: 모바일 세로

**목표:** 스마트폰 배경  
**권장 해상도:** 832×1216

```
이유:
  ✅ 9:16 비율과 유사
  ✅ 모바일 화면 최적
  ✅ 인물 촬영 적합
```

---

#### Scenario 4: 인쇄물

**목표:** A4 크기 (210×297mm)  
**권장 해상도:** 896×1152 또는 832×1216

```
이유:
  ✅ 세로 비율 유사
  ✅ 고해상도 출력 가능
  ✅ 품질 보장
```

---

### ⚠️ 주의사항

#### 1️⃣ 해상도 크게 변경될 수 있음

```
입력: 500×800
조정: 640×1536 (3배 확대!)

문제점:
  ⚠️ 원본 구도와 차이
  ⚠️ 예상과 다른 결과
```

**해결:**
```
✅ 가까운 권장 해상도로 직접 입력
✅ 비활성화 후 정확한 해상도 사용
```

#### 2️⃣ 정확한 해상도 필요 시

```
특수 작업:
  - 정확한 픽셀 크기 필요
  - 특정 비율 유지
  - 후처리 파이프라인

해결:
  ✅ Nearest SDXL Resolution 비활성화
  ⚠️ 품질 저하 감수
```

#### 3️⃣ Upscaler 활용

```
과정:
  1. SDXL로 권장 해상도 생성
  2. Upscaler로 원하는 크기 확대

장점:
  ✅ 최적 품질 유지
  ✅ 정확한 최종 크기
  ✅ 아티팩트 최소화
```

### 💡 최적화 팁

**1. 비율 먼저 결정**

```

```

