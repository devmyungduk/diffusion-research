# Sampler Comparison

노이즈 제거 방식 및 스텝 수에 따른 특성 비교

---

## 🎯 가이드 개요

**대상 독자:** "어떤 Sampler를 써야 할지" 고민되는 사용자
**사전 이해:** KSampler 노드 사용 경험
**예상 소요 시간:** 15분

**핵심 내용:**
- Sampler(그리는 방식)와 Scheduler(속도 조절)의 차이
- Euler, DPM++, Ancestral 등 주요 알고리즘의 특징
- 실사/애니메이션 등 상황별 최적의 조합 추천

## 샘플러별 특성

| 샘플러 | 방식 | 장점 | 단점 | 수렴 속도 |
|:-------|:-----|:-----|:-----|:----------|
| Euler a | 고전/빠른 수렴 | 속도 | 디테일 손실 | ████████░░ |
| DPM++ 2M Karras | 최신/안정 | 품질 | 계산량 | ██████░░░░ |

## 샘플링 과정 비교

### Euler a (빠른 단계)
```
Noise (t=1000)
    │ step size: large
    ├─> t=750  ████░░░░░░  (큰 점프)
    ├─> t=500  ██████░░░░  
    ├─> t=250  ████████░░
    └─> t=0    ██████████  (완료)

특징: 적은 step으로 빠른 수렴, 디테일 희생
```

### DPM++ 2M Karras (안정적)
```
Noise (t=1000)
    │ step size: adaptive (Karras schedule)
    ├─> t=900  ██░░░░░░░░  (작은 시작)
    ├─> t=700  ████░░░░░░
    ├─> t=400  ███████░░░  (중간 가속)
    ├─> t=150  █████████░
    └─> t=0    ██████████  (완료)

특징: 2차 미분 활용, 품질-속도 균형
```

### Heun (정밀)
```
Noise (t=1000)
    │ step size: very small + correction
    ├─> t=950  █░░░░░░░░░  (예측)
    │   └─> correction     (보정)
    ├─> t=900  ██░░░░░░░░
    ├─> t=850  ███░░░░░░░  ...
    └─> t=0    ██████████  (완료)

특징: 각 step마다 예측-수정, 높은 정확도
```

### DDIM (결정적)
```
Noise (t=1000)
    │ deterministic path
    ├─> t=800  ██░░░░░░░░  (고정 경로)
    ├─> t=600  ████░░░░░░
    ├─> t=400  ██████░░░░
    ├─> t=200  ████████░░
    └─> t=0    ██████████  (완료)

특징: 동일 seed → 동일 결과 보장
```

## 가중치 표기법

### 🎯 프롬프트 가중치란?

**프롬프트 가중치 = 특정 개념의 중요도를 조절하는 방법**

`(개념:가중치)` 형식으로 AI에게 강조할 부분을 알려줍니다.

### 📊 문법

```
(개념:가중치)

예시: (ink and painting:1.5)
```

| 요소 | 설명 | 예시 |
|------|------|------|
| **개념** | 강조할 단어/스타일 | ink and painting |
| **:** | 구분자 | : |
| **가중치** | 강도 (0.1~2.0) | 1.5 |

### 💡 가중치 범위

| 가중치 | 의미 | 효과 | 사용 시기 |
|--------|------|------|-----------|
| **0.1~0.5** | 최소값 | 개념 거의 무시 | 약하게 힌트만 |
| **0.8** | 약함 | 은은하게 적용 | 부드러운 표현 |
| **1.0** | 기본값 | 표준 강도 | 일반적인 경우 |
| **1.5** | 강함 | 명확하게 강조 | 중요한 요소 |
| **2.0** | 최대값 (권장) | 매우 강조 | 핵심 스타일 |
| **2.5+** | ⚠️ 과도 | 이미지 왜곡 가능 | 비권장 |

### 🔄 실전 예시

#### 예시 1: 스타일 강조

```
(ink and painting:1.5), (watercolor:0.8), portrait

결과:
  잉크 페인팅 스타일: 150% 강조
  수채화 요소: 80% (약하게)
  인물 초상화: 100% (기본)
```

#### 예시 2: 다중 개념 조절

```
(masterpiece:1.3), (detailed:1.2), (photo realistic:1.5), portrait of a woman

결과:
  각 요소의 강도를 세밀하게 제어
  품질 + 디테일 + 사실성 모두 강조
```

#### 예시 3: 색상 제어

```
(red apple:1.5), (blue background:1.3), on a wooden table

결과:
  빨간 사과: 강하게 강조
  파란 배경: 중간 강조
  나무 테이블: 기본 강도
```

### ⚙️ 작동 원리

**AI 이미지 생성 과정에서:**

```
프롬프트: (ink and painting:1.5)
    ↓
CLIP 인코더가 해석
    ↓
"ink and painting" 개념의 중요도 50% 증가
    ↓
생성 과정에서 해당 스타일 더 강하게 반영
```

### 💡 사용 이유

| 목적 | 설명 | 예시 |
|------|------|------|
| **세밀한 제어** | 특정 요소 강도 미세 조정 | 색상, 스타일 정밀 제어 |
| **스타일 강조** | 원하는 느낌 강화 | (oil painting:1.5) |
| **균형 조정** | 여러 요소의 상대적 비율 | 배경 vs 전경 |
| **결과 최적화** | 반복 실험으로 최적값 찾기 | A/B 테스트 |

### ⚠️ 주의사항

**1. 과도한 가중치 피하기**

```
❌ (red:3.0) → 색상 왜곡, 아티팩트
✅ (red:1.5) → 자연스러운 강조
```

**2. 균형 유지**

```
❌ 모든 단어에 2.0 → 의미 없음
✅ 중요한 것만 강조 → 효과적
```

**3. 실험과 반복**

```
1.0 → 1.2 → 1.5 → 1.3 (최적)
단계적으로 조정하며 최적값 탐색
```
